# スイーツパニック ゲーム仕様書

## 1. 概要

`SWEETS PANIC` は、4種類のケーキキャラクターを操作して敵ウェーブとボスを倒し続ける、協力型トップダウンアクションゲームである。主軸は「跳ね返して、倒す」。壁や障害物で弾・突進・敵弾を反射させるほど攻撃が強くなり、反射を狙うことがスコア、攻略、キャラ性能のすべてに結びつく。

DX11版では3Dトップダウン視点で表現する。ステージ、キャラクター、敵、弾、アイテムは3Dモデルへ差し替え可能な構成を前提とし、初期実装はプリミティブメッシュとHLSLシェーダーで成立させる。

## 2. ゲームコンセプト

- ジャンル: 協力トップダウンアクション + 弾幕シューティング要素
- プレイ人数: 1から4人
- プレイヤー構成: 人間プレイヤーとAI操作を混在可能
- 目的: ウェーブごとに出現する敵を倒し、3ウェーブごとのボスを撃破しながらエンドレスにスコアを伸ばす
- 重要な遊び:
  - 通常攻撃、チャージ攻撃、必殺技を使い分ける
  - 反射地形を使って攻撃の威力とスコアを伸ばす
  - 弾幕を集中移動、グレイズ、ボムでしのぐ
  - キャラごとの役割を組み合わせて協力する

## 3. コアループ

1. ウェーブ開始時にステージ地形と敵編成が決まる。
2. プレイヤーは敵を倒し、EXP、スコア、必殺ゲージ、ドロップを得る。
3. レベルアップ時に汎用強化を選ぶ。
4. ボス撃破時にキャラ固有スキルを選ぶ。
5. 次ウェーブへ進み、敵の量、攻撃密度、報酬量が増える。
6. 全員が戦闘不能になるまで周回を続ける。

## 4. 操作仕様

| 入力 | アクション |
| --- | --- |
| WASD / 方向キー | 移動 |
| Shift | 集中移動。移動速度低下、自機当たり判定表示 |
| マウス移動 | 照準 |
| 左クリック / Space | 通常攻撃 |
| 左クリック長押し | チャージ攻撃 |
| 右クリック | キャラ固有サブ操作。チーズは壁設置 |
| Q | 必殺技 |
| X / Control | ボム。敵弾消去、短時間無敵、周囲へダメージ |
| P / Esc | ポーズ |
| Enter | タイトル開始 / ゲームオーバー後リスタート |

複数人プレイでは、1Pはキーボード+マウスを標準とする。2P以降はゲームパッドを標準入力とし、未接続枠はAI操作にできる。

## 5. プレイアブルキャラクター

内部分類は `CharacterType` とする。

| CharacterType | 表示名 | 役割 | 通常攻撃 | チャージ攻撃 | 必殺技 | 反射の強み |
| --- | --- | --- | --- | --- | --- | --- |
| `Shortcake` | ショート | 連射 / 追尾 | 追尾するイチゴ弾を連射 | 命中時に5発へ分裂する大きなイチゴ弾 | 巨大メテオを指定地点に落とす | 追尾弾が壁反射後も敵を追い、反射回数で火力が伸びる |
| `Chocolate` | チョコ | 近接 / 範囲 | 大剣の回転薙ぎ | 前方へ貫通する斬撃波 | 高速回転の時計斬り | 壁際や障害物裏の敵を斬撃波でまとめて制圧する |
| `Cheese` | チーズ | 防御 / 反射補助 | 重いチーズ弾 | トゲ付きの動く壁を出す | 無敵要塞を展開する | 設置壁が敵弾を受け止め、チーズ弾として反射する |
| `Roll` | ロール | 制御 / 反射 | 壁で跳ね回るロール弾 | 自分が転がって突進 | 全画面叩きつけ | 弾も突進も複数回反射し、連続ヒットを狙える |

各キャラは性能プリセット `LoadoutPreset` を持つ。基本項目は最大HP、移動速度、攻撃倍率、クールダウン倍率、初期必殺ゲージ、役割説明、表示色とする。

## 6. 攻撃システム

### 通常攻撃

通常攻撃は低リスクで連続使用できる基本攻撃である。武器定義 `WeaponDef` は、クールダウン、速度、ダメージ、半径、貫通数、反射回数を持つ。

### チャージ攻撃

チャージ攻撃は通常攻撃ボタン長押しで発動する。チャージ中は移動可能だが、発動までに隙がある。キャラごとに通常攻撃と違う役割を持たせる。

- ショート: 大弾。命中時に5方向へ分裂
- チョコ: 貫通斬撃波
- チーズ: トゲ付き移動壁。敵と敵弾を押し返す
- ロール: プレイヤー自身の突進。壁で跳ね返り連続ヒット

### 必殺技

必殺ゲージ100で `Q` により発動する。敵撃破、グレイズ、アイテム取得、ウェーブクリアでゲージが増える。

必殺技は画面を大きく変える高火力または強防御効果を持つ。発動中は短時間の無敵または被弾猶予を付与する。

## 7. 反射システム

反射は本作の中心ルールである。弾、突進、敵弾、チーズ壁など、複数の攻撃要素が壁や障害物で跳ね返る。

### 反射対象

- プレイヤー弾
- ロールの突進
- チーズ壁で受け止めた敵弾
- 一部の敵弾
- ステージギミックで弾かれた攻撃

### 反射倍率

反射するたびに攻撃の威力を上げる。基準倍率は1反射ごとに1.5倍、最大約5倍とする。倍率は弾データに反射回数として保持する。

### 反射キル

反射済み攻撃で敵にとどめを刺した場合、スコアを2倍にする。演出として反射キル用の発光、効果音、スコア表示を出す。

### 反射必須敵

鏡属性の敵は通常攻撃だけでは倒せず、反射済み攻撃でのみ有効ダメージを受ける。攻略順を作るため、通常ウェーブ中盤以降から出現させる。

## 8. 東方風シューティング要素

本作はアクションゲームを基礎にしつつ、ボス戦では弾幕シューティングの緊張感を入れる。

### 集中移動

`Shift` を押している間、移動速度を下げる。細かい回避を可能にし、自機の小さい当たり判定を表示する。

### 自機当たり判定

プレイヤーの見た目半径とは別に、被弾判定用の小さい `hitboxRadius` を持つ。弾幕との接触判定はこの半径を使う。

### グレイズ

敵弾が自機の当たり判定には触れず、外側のグレイズ範囲を通過した場合にグレイズ成立とする。グレイズ時はスコア、必殺ゲージ、連続グレイズ数が増える。

### ボム

`X` または `Control` でボムを発動する。ボムは残数制で、効果は以下とする。

- 画面上または周囲の敵弾を消す
- 短時間無敵になる
- 周囲の敵とボスへダメージを与える
- 消した弾数に応じてスコアを得る

### ボス弾幕

ボスはHP割合でフェーズが進み、弾幕が増える。

- 放射弾: 全方向へ等間隔に弾を発射
- 狙い撃ち: プレイヤー方向へ扇状に弾を発射
- 螺旋弾: 回転角度を持つ連続弾
- 曲がる弾: `angularVel` で軌道を曲げる
- 加減速弾: `accel` で速度を変化させる

## 9. ステージ仕様

内部分類は `StageType` とする。

| StageType | 表示名 | 地形仕様 | 狙い |
| --- | --- | --- | --- |
| `Donut` | ドーナツ | 中央に大きな島 | 外周反射と中央遮蔽を使う |
| `TwinIsland` | 双子島 | 2つの島を配置 | 島の間を通す射線管理 |
| `Pinball` | ピンボール | バンパーが弾を強く弾く | 反射コンボを作りやすい |
| `RingCorridor` | リング回廊 | 内側と外側の二層構造 | 接近戦と遠距離戦を分ける |
| `FourPillars` | 四隅の柱 | 対称配置の4本柱 | 協力時の担当レーンを作る |
| `MovingIsland` | 動く島 | 障害物がゆっくり移動 | 反射角が時間で変化する |
| `ShrinkRing` | 収縮リング | 安全地帯が徐々に狭まる | 長期戦を防ぎ圧力を作る |
| `BossArena` | ボス戦円形 | 開けた円形フィールド | 弾幕回避を見やすくする |

全ステージの外周、島、柱、バンパーは反射対象である。ボス戦は視認性を優先し、基本は `BossArena` を使う。

## 10. 敵仕様

内部分類は `EnemyType` とする。

| EnemyType | 役割 | 行動 | 対策 |
| --- | --- | --- | --- |
| `Normal` | 基本ザコ | プレイヤーへ接近 | 通常攻撃で処理 |
| `Shield` | 前方防御 | 正面からのダメージを軽減 | 背後か反射弾で攻撃 |
| `Split` | 物量 | 撃破時に小型2体へ分裂 | 範囲攻撃でまとめて処理 |
| `Healer` | 支援 | 周囲の敵を回復 | 優先撃破 |
| `Barrier` | 支援 | 周囲の敵を短時間無敵化 | 反射や突進で分断 |
| `Mirror` | 反射必須 | 通常弾を無効化 | 反射済み攻撃で撃破 |
| `Mine` | かく乱 | 接近または撃破で爆発 | 離れて処理 |
| `Teleport` | かく乱 | 位置を変えながら射撃 | 予測して範囲攻撃 |

敵はウェーブが進むほどHP、攻撃力、速度、出現数、射撃密度が上がる。

## 11. ボス仕様

内部分類は `BossType` とする。

| BossType | 表示名 | 主な行動 | 特徴 |
| --- | --- | --- | --- |
| `Demon` | 大ボス | 放射弾、狙い撃ち、召喚 | 標準ボス |
| `DonutKing` | ドーナツキング | 大量召喚、低速弾幕 | 物量で押す |
| `MirrorMacaron` | ミラーマカロン | 分身、本体隠し、反射要求 | 見極めと反射が必要 |
| `GravityPudding` | グラビティプリン | 引き寄せ、吹き飛ばし、曲がる弾 | 位置取りを崩す |
| `TerritoryCake` | テリトリーケーキ | 床ダメージ、領域制圧 | 安全地帯管理 |

ボスHPはフェーズ制にする。75%、50%、25%を下回るごとに攻撃密度と攻撃パターンを増やす。

## 12. 成長仕様

### レベルアップ

敵撃破でEXPを得る。一定値でレベルアップし、汎用スキルを選択する。

汎用スキル例:

- 最大HP上昇
- 攻撃力上昇
- 連射速度上昇
- 移動速度上昇
- 弾サイズ上昇
- 反射回数増加
- 必殺ゲージ増加量上昇

### キャラ固有スキル

大ボス撃破後、各キャラ専用の強化を選べる。全20種を基準とし、1キャラ5種ずつ用意する。

例:

- ショート: 追尾性能強化、分裂数増加、反射後加速
- チョコ: 斬撃範囲拡大、斬撃波多段化、近接中ダメージ軽減
- チーズ: 壁耐久上昇、壁反射倍率上昇、壁の自動展開
- ロール: 反射回数増加、突進後無敵、弾の再加速

### フィーバー

連続撃破でフィーバーゲージが上がる。一定時間、スコア倍率とドロップ率を上げる。被弾または一定時間撃破なしで低下する。

### 合体必殺

複数プレイヤーの必殺ゲージが揃っている場合、近接状態で合体必殺を発動できる。発動キャラの組み合わせで効果を変える。

### コア育成

ドロップするケーキコアで弾性能を永続強化する。強化候補は「よく弾む」「貫通する」「反射後加速」「敵弾消し」など。

## 13. アイテム仕様

内部分類は `PickupType` とする。

| PickupType | 効果 |
| --- | --- |
| `Attack` | 一定時間、攻撃力と連射性能を上げる |
| `Slow` | 一定時間、敵と敵弾を遅くする |
| `Invincible` | 一定時間、無敵になる |
| `Magnet` | アイテムを自動引き寄せする |
| `BombDamage` | 全画面または広範囲へ大ダメージ |
| `Heal` | HPを回復する |
| `UltFull` | 必殺ゲージを最大にする |
| `Spread` | 一定時間、ショットを拡散化する |
| `Speed` | 一定時間、移動速度を上げる |
| `ScoreDouble` | 一定時間、スコア倍率を2倍にする |

低確率ブーストアイテムは視覚的に強く光らせ、取得価値が直感的にわかるようにする。

## 14. UI仕様

### タイトル / キャラ選択

自機性能選択は、直感的な4キャラカードUIにする。

各カードに表示する要素:

- キャラ名とケーキ種別
- 役割アイコン
- 通常攻撃、チャージ攻撃、必殺技の短い説明
- HP、速度、攻撃、連射、反射性能のステータスバー
- 操作プレビューまたは小さな攻撃アニメーション
- 人間操作 / AI操作の切り替え

選択中カードはサイズ、枠線、足元リング、ステータスバーの明度で強調する。文章を読まなくても、速いキャラ、硬いキャラ、反射向きキャラが判別できることを優先する。

### 戦闘HUD

- HPバー
- 必殺ゲージ
- ボム残数
- レベルとEXP
- スコア
- ウェーブ数
- グレイズ数
- フィーバー状態
- ボスHPバー
- 反射キル / レベルアップ / アイテム取得メッセージ

### チュートリアル

9枚のチュートリアルスライドで、移動、攻撃、チャージ、必殺、反射、グレイズ、ボム、救助、成長を説明する。

## 15. 協力プレイ仕様

- 最大4人まで参加可能
- 各枠は人間操作またはAI操作を選べる
- ダウンした仲間は近づいて救助できる
- 救助中は行動制限があり、成功まで一定時間が必要
- 全員がダウンするとゲームオーバー
- 1人でも生存していればウェーブは継続する

AIは以下を優先する。

1. 敵弾回避
2. ダウン仲間の救助
3. ヒーラー、バリア、鏡など優先敵の撃破
4. ボス攻撃
5. アイテム取得

## 16. 勝敗条件

本作はエンドレス・スコアアタックであるため、明確なクリアはない。

- ウェーブクリア: 敵とボスを全滅させる
- ゲームオーバー: 全プレイヤーが戦闘不能になる
- 評価軸: 到達ウェーブ、スコア、反射キル数、グレイズ数、被弾回数、救助回数
